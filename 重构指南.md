# weixin-agent-sdk 重构指南

## 重构目标

将本项目从 TypeScript/Node.js 完整重构为 **Python** 实现，同时在重构过程中着重提升以下两个维度：

- **性能**：利用 Python 异步生态（`asyncio` + `aiohttp`）实现高并发长轮询；媒体处理、CDN 上传等 I/O 密集操作全部异步化；避免阻塞主线程。
- **鲁棒性**：完善错误处理与自动重连机制；对会话过期、网络抖动、CDN 超时等场景做明确的重试与降级策略；关键状态持久化，确保重启后无缝续传。

重构后的项目应保持与原 SDK 相同的对外接口语义（`login` / `start` / `Agent`），使现有业务逻辑可以平滑迁移。

---

## 项目概述

本项目是一个**微信 AI Agent 桥接框架**，让任意 AI 后端可以通过微信接收和发送消息。代码由 `@tencent-weixin/openclaw-weixin` 改造而来，仅供学习交流使用。

### Agent 接口定义

```typescript
interface Agent {
  chat(request: ChatRequest): Promise<ChatResponse>;
}

interface ChatRequest {
  conversationId: string;   // 用户标识，可用于维护多轮对话
  text: string;             // 文本内容
  media?: {                 // 附件（图片/语音/视频/文件）
    type: "image" | "audio" | "video" | "file";
    filePath: string;       // 本地文件路径（已下载解密）
    mimeType: string;
    fileName?: string;
  };
}

interface ChatResponse {
  text?: string;            // 回复文本（支持 markdown，发送前自动转纯文本）
  media?: {                 // 回复媒体
    type: "image" | "video" | "file";
    url: string;            // 本地路径或 HTTPS URL
    fileName?: string;
  };
}
```

## 核心数据流

```
微信用户发消息
    ↓
getUpdates 长轮询 (monitor/monitor.ts)
    ↓
解析消息 + 下载解密媒体 (messaging/process-message.ts)
    ↓
调用 agent.chat(request)
    ↓
发送回复 (messaging/send.ts / send-media.ts)
    ↓
上传媒体到微信 CDN (cdn/cdn-upload.ts)
    ↓
微信用户收到回复
```

## 技术细节

- 使用**长轮询** (`getUpdates`) 接收消息，无需公网服务器
- 媒体文件通过微信 CDN 中转，**AES-128-ECB** 加密传输
- 单账号模式：每次 `login` 覆盖之前的账号
- 断点续传：`get_updates_buf` 持久化到 `~/.openclaw/`，重启后从上次位置继续
- 会话过期自动重连（errcode -14 触发 1 小时冷却后恢复）
- 原 TS 版本运行时要求：Node.js >= 22（Python 重构版本不适用）

## 支持的消息类型

### 接收（微信 → Agent）

| 类型 | `media.type` | 说明 |
|------|-------------|------|
| 文本 | — | `request.text` 直接拿到文字 |
| 图片 | `image` | 自动从 CDN 下载解密，`filePath` 指向本地文件 |
| 语音 | `audio` | SILK 格式自动转 WAV（需安装 `silk-wasm`） |
| 视频 | `video` | 自动下载解密 |
| 文件 | `file` | 自动下载解密，保留原始文件名 |
| 引用消息 | — | 被引用文本拼入 `request.text`，被引用媒体作为 `media` 传入 |
| 语音转文字 | — | 微信侧转写的文字直接作为 `request.text` |

### 发送（Agent → 微信）

| 类型 | 用法 |
|------|------|
| 文本 | 返回 `{ text: "..." }` |
| 图片 | 返回 `{ media: { type: "image", url: "/path/to/img.png" } }` |
| 视频 | 返回 `{ media: { type: "video", url: "/path/to/video.mp4" } }` |
| 文件 | 返回 `{ media: { type: "file", url: "/path/to/doc.pdf" } }` |
| 文本 + 媒体 | `text` 和 `media` 同时返回，文本作为附带说明发送 |
| 远程图片 | `url` 填 HTTPS 链接，SDK 自动下载后上传到微信 CDN |

### 关键技术选型

| TS 原实现 | Python 替代方案 | 说明 |
|-----------|----------------|------|
| `node-fetch` / 原生 fetch | `aiohttp` | 异步 HTTP，支持连接池与超时控制 |
| `silk-wasm` | `pysilk` | SILK 音频格式解码，纯 Python 绑定 |
| AES-128-ECB（Node crypto） | `pycryptodome` (`Crypto.Cipher.AES`) | 图片解密，接口一致 |
| `qrcode-terminal` | `qrcode` + `rich` | 终端二维码渲染 |
| JSON 文件持久化 | 标准库 `json` + `pathlib` | 替代 `sync-buf.ts` 的状态持久化 |
| TypeScript `interface` | `typing.Protocol` / `dataclasses` | Agent 抽象接口定义 |
| `AbortSignal` | `asyncio.Event` | 优雅停止信号 |
| pnpm monorepo | `uv` workspace 或单包结构 | Python 包管理推荐使用 `uv` |

### Agent 接口（Python 版）

```python
from typing import Literal, Protocol
from dataclasses import dataclass

@dataclass
class MediaAttachment:
    type: Literal["image", "audio", "video", "file"]
    file_path: str
    mime_type: str
    file_name: str | None = None

@dataclass
class ChatRequest:
    conversation_id: str
    text: str
    media: MediaAttachment | None = None

@dataclass
class MediaReply:
    type: Literal["image", "video", "file"]
    url: str
    file_name: str | None = None

@dataclass
class ChatResponse:
    text: str | None = None
    media: MediaReply | None = None

class Agent(Protocol):
    async def chat(self, request: ChatRequest) -> ChatResponse: ...
```

### 鲁棒性设计要点

- **重试策略**：对网络 I/O（HTTP 请求、CDN 上传）使用指数退避重试，推荐 `tenacity` 库
- **连接池**：`aiohttp.ClientSession` 全局复用，避免频繁建立 TCP 连接
- **会话过期**：errcode -14 捕获后进入冷却协程（`asyncio.sleep`），不阻塞事件循环
- **并发隔离**：每条消息的 agent.chat 以独立 `asyncio.Task` 运行，避免单条慢响应阻塞后续消息
- **状态持久化**：断点续传 buf 写入时使用原子替换（写临时文件再 rename），防止进程崩溃导致数据损坏
- **类型安全**：全量使用 `dataclasses` + `typing`，可选引入 `pydantic` 做边界校验
